19/*
20 Cache System:
21
22 The goal of this cache system is to be something that supports the engine for the long term. The chosen paradigm is that of a refcount-less cache, which accumulates
23 items either manually, or from pre loaded files. The public API supports getting resources from the cache (if existant) through functions like `ye_image` and `ye_font`.
24 There are also functions for manually adding resources to the cache, such as `ye_create_texture` and `ye_create_font`.
25 The cache is only explicitely cleared when ye_clear_*_cache is called, which will free all cached resources of specified type.
26 Normally, the scene manager will just clear the texture cache between scenes. The font and color cache will be cleared when the engine is shut down.
27
28 This cache paradigm suits the intended use of the engine, as I envision loading all scene assets up front, and then clearing them when the scene is unloaded.
29 To minimize the overhead of loading textures that might not need to be cleared on scene switch, `ye_pre_cache_scene` will reuse duplicate textures that have
30 already been cached when switching scenes.
31
32 Pre caching a scene will be opt-in, and will be done by the scene manager. In the case of very large scenes, the developer can still load textures on demand,
33 but will still need to find a way to pre-cache all their fonts and colors as needed.
34
35 The low level api does allow for deletion of individual resources, but I'm only including this for edge case compatibility, as I wont be using it for the purpose of
36 my games (yet).
37
38 The cache is implemented using UTHASH for simplicity and
39
40 The cache is very opt in, as you are only required to pass the pointers to resources when constructing ECS
41 components, meaning you could forgo the cache entirely and manually manage your memory. This is not recommended.
42
43 Tracking cache size:
44
45 This could be pretty easily done, but its not strictly necessary and will add overhead (unless optimized out)
46 It could be done like this if ever needed:
47
48 int ye_texture_size(SDL_Texture *texture){
49 int w, h;
50 SDL_QueryTexture(texture, NULL, NULL, &w, &h);
51 return w * h * 4;
52 }
53
54 TODO:
55 - destruction of individual cache items
56 - caching of scene files
57
58 SHAPING:
59 scene.yoyo files can specify a styles.yoyo that it is dependant on... which scene manager can auto load
60*/
